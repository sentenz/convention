    1.3. Code Analysis¶

    Code Check

    • clang-tidy tool and doc

      Configure .clang-tidy in the project.

    • Cpplint

      • Detect style errors, see cpplint.py tool.
      • Configure CPPLINT.cfg in the root project.
      • False positives can be ignored by putting // NOLINT at the end of the line or // NOLINTNEXTLINE in the previous line.

    • scan-build

      Tool for clang-analyzer.

    • Cppcheck

      • Configure cppcheck-suppressions.txt in the project.
      • See manual.

    • Infer

      A static analyzer for Java, C, C++, and Objective-C.

    Code Format

    • clang-format

      Configure in the root project.

      • .clang-format
      • .clang-format-ignore
      • Use clang-format-configurator to modify a .clang-format file.

    • Valgrind
